As perguntas
que todo mundo
tem medo de fazer.

Antes de qualquer ferramenta, as pessoas precisam confiar. Aqui estão as respostas honestas: baseadas nas políticas reais do plano Teams.

Meus dados e briefings vão ser usados pra treinar a IA?
+
Não. No plano Teams: que é o que a LR vai adotar: a Anthropic garante contratualmente que conversas e documentos não são usados para treinar modelos. O que fica na agência, fica na agência. Isso é diferente do plano gratuito.
A Anthropic pode acessar o que a gente conversa com o Claude?
+
No Teams, o acesso humano às conversas é extremamente restrito: ocorre apenas em casos específicos de segurança e com procedimentos formais. Não é uma empresa monitorando seu chat em tempo real.
E se eu colar o briefing do cliente lá? Vai vazar?
+
O risco é equivalente ao de mandar um e-mail pelo Gmail ou salvar no Google Drive: que a agência já usa. O dado trafega encriptado. No Teams, o contrato protege. O único cenário de risco real é usar o plano gratuito pessoal com dados de cliente.
E se alguém usar o plano gratuito no computador pessoal?
+
Esse é o cenário de maior risco real: e é exatamente por isso que a implementação oficial existe. Com a assinatura Teams da agência, todo mundo usa pela conta corporativa, com proteção contratual. A conta Teams é a solução.
A IA pode cometer erros e prejudicar um cliente?
+
Pode errar: como qualquer pessoa da equipe. O Claude costuma sinalizar quando está inseguro. A responsabilidade final é sempre humana: a IA é uma camada de apoio, não de decisão. Nenhum texto vai pro cliente sem passar pelos olhos da equipe.
Isso não vai substituir o trabalho das pessoas?
+
A experiência de agências que já implementaram mostra o oposto: a equipe passa menos tempo em tarefas repetitivas e mais em criação e estratégia. Quem vai ser substituída, mais cedo ou mais tarde, são as agências que não adotarem.

O que é um Live Artifact

É um artefato que fica conectado a uma fonte de dados: nesse caso, o seu Gmail. Você não fica preso num chat que some. Você cria o artefato uma vez, salva, e toda vez que abre ele executa a busca com dados frescos. É a diferença entre tirar uma foto e instalar uma câmera.

2

Cole o prompt de criação do artefato abaixo. O Claude vai gerar o dashboard interativo e perguntar se você quer salvá-lo como Live Artifact.

3

Confirme para salvar. A partir daí, o artefato aparece na sua lista de artefatos e você abre quando quiser: ele sempre puxa os dados mais recentes.

Todos os termos técnicos do universo Claude em linguagem de agência. Sem jargão desnecessário, sem rodeio. Quando alguém falar um desses termos numa reunião, você já sabe do que se trata.

A
Agente Agent
Um Claude configurado para tomar ações de forma autônoma: acessar ferramentas, executar tarefas em sequência, tomar decisões sem você aprovar cada passo. É diferente de um chat normal: o agente age, não só responde. Requer API ou Claude Code para configurar. Ex: um agente que toda semana puxa os dados do Sheets, gera o relatório e envia por e-mail sozinho.
Avançado
API
A porta dos fundos do Claude: uma forma de acessá-lo direto via código, sem passar pelo site. Permite integrar o Claude dentro de outros sistemas, automatizar fluxos e criar ferramentas próprias. O plano Teams não inclui API por padrão. Ex: um sistema interno que usa a API do Claude para classificar os e-mails dos clientes automaticamente.
Avançado
Artefato Artifact
Quando o Claude gera um arquivo de verdade: um documento, uma tabela, um código, uma apresentação: em vez de só texto numa mensagem. Aparece num painel lateral separado e pode ser exportado. Ex: pedir ao Claude para "criar uma tabela de calendário de conteúdo" gera um artefato que você pode baixar como CSV.
Básico
C
Claude Code
Uma versão do Claude que funciona pelo terminal do computador: para pessoas que sabem programar. Ele lê, escreve e executa código diretamente nos arquivos do projeto. Para a LR, é útil só se houver alguém técnico na equipe. Ex: pedir ao Claude Code para "analisar todos os arquivos HTML do site e corrigir os links quebrados".
AvançadoDev
Claude Design
Produto da Anthropic para criar peças visuais por conversa: layouts, apresentações, assets de social. Você descreve, ele constrói, você refina. Exporta para Canva, PPTX ou PDF. BásicoNovo
Claude Cowork
Uma versão do Claude.ai focada em trabalhar com arquivos: você arrasta PDFs, imagens, planilhas e o Claude lê, analisa e extrai informação. Tem mais capacidade de processar documentos grandes do que o chat normal. Ex: arrastar a NF de um fornecedor e pedir para o Claude extrair os dados em tabela.
Básico

Uma integração que conecta o Claude a uma ferramenta externa: Google Drive, Gmail, Figma, Adobe, Slack. Ex: se você colar um briefing no início da conversa, ele está "no contexto" e o Claude pode referenciá-lo em qualquer resposta seguinte.

Ex: a "Skill: Redator LR" é um Claude que já sabe o estilo da agência, os clientes e o que nunca aprovar.

O que é uma Skill na prática

Uma Skill é um Projeto no Claude com instruções detalhadas que transformam o Claude em um assistente especializado. A diferença de usar o Claude normal vs usar uma Skill é a diferença entre pedir direção a um passante vs pedir direção a um guia que conhece cada rua da cidade.

Claude especializado

O contexto já está configurado. O resultado é consistente. Toda a equipe trabalha com o mesmo assistente.

Como criar uma Skill para a LR

Tecnicamente é um Projeto no Claude com instruções bem escritas. Mas a qualidade das instruções é o que separa uma Skill boa de uma inútil.

01

Crie um Projeto novo no Claude

Vá em Projetos → Novo Projeto. Nomeie de forma clara: "Skill: Briefing LR", "Skill: Redação Agroceres", etc. O nome ajuda a equipe a encontrar e saber o que esperar.

02

Escreva as Instruções do Projeto

É aqui que a mágica acontece. As instruções definem quem o Claude é nesse projeto, o que ele sabe, como ele responde e o que ele nunca faz. Use o template abaixo como base.

03

Suba os documentos de referência

Cole PDFs, documentos e exemplos reais no projeto. O Claude usa esses arquivos como fonte de verdade. Quanto mais material relevante, mais precisa a Skill fica.

04

Teste antes de compartilhar

Abra o projeto e faça perguntas que a equipe faria. Veja se as respostas estão no tom certo, se ele usa o vocabulário correto, se não inventa informação. Ajuste as instruções se necessário.

05

Compartilhe com a equipe via Teams

No plano Teams, você pode dar acesso ao Projeto para toda a equipe ou para grupos específicos. A Skill fica disponível para todos: e você mantém controle sobre as instruções.

Limites & Tokens

Por que o Claude
"parou de
responder"?

Tokens, limites de uso, janela de contexto: parece técnico mas é simples. Entender isso evita frustração e ajuda a usar o Claude de forma mais inteligente.

O que é um token: sem complicação

🔤
Token = pedaço de texto

~1 token por palavra

Na prática, 1 token equivale a aproximadamente ¾ de uma palavra em português. "Marketing" = 2 tokens. "Ok" = 1 token. É a unidade que o Claude usa para ler e escrever.

📥
Tokens de entrada

O que você manda

Tudo que você escreve no chat: seu prompt, o texto colado, o documento enviado: consome tokens de entrada. Um briefing de 2 páginas são cerca de 800–1000 tokens.

📤
Tokens de saída

O que o Claude responde

Cada palavra que o Claude escreve de volta também consome tokens. Uma resposta longa (tipo uma campanha completa) pode usar 1.500–2.000 tokens.

Referência rápida: uma conversa típica de atendimento (briefing + resposta) usa cerca de 2.000–4.000 tokens. Um calendário de conteúdo completo pode usar 5.000–8.000 tokens. O plano Teams tem um volume mensal generoso para uso profissional: você não vai "acabar" fazendo uso normal.

Janela de contexto: o que o Claude "lembra"

Este é o conceito mais importante e o mais mal entendido. O Claude não tem memória permanente entre conversas: ele só "lembra" do que está na conversa atual. Essa conversa tem um limite de tamanho chamado janela de contexto.

📏

O que é a janela de contexto

Pensa como uma folha de papel muito grande. Tudo que está nessa folha: suas mensagens e as respostas do Claude: é o que ele "vê" e considera. Quando a folha enche, o Claude começa a "esquecer" o início da conversa.

⚠️

Quando a conversa fica longa demais

Se você está numa conversa muito longa e o Claude começa a responder de forma inconsistente ou "esquece" o que foi dito no início: é sinal de que a janela de contexto está chegando ao limite. A solução é simples: comece uma nova conversa.

📁

Por que Projetos resolvem isso

O Projeto mantém as Instruções e os Documentos sempre disponíveis: mesmo em conversas novas. Então você pode começar uma nova conversa sem perder o contexto do cliente ou da tarefa.

🧠

Sem memória entre conversas diferentes

O Claude não lembra da conversa de ontem. Cada nova conversa começa do zero: a menos que esteja num Projeto com contexto configurado. Por isso os Perfis de Cliente são tão importantes.

Limites do plano Teams: o que acontece na prática

O plano Teams tem um limite de uso por período. Quando você atinge esse limite, o Claude não para de funcionar: ele muda para um modelo mais leve temporariamente, ou pede para você esperar alguns minutos.

Como usar de forma eficiente

Não é sobre economizar por medo de gastar. É sobre usar bem para conseguir o melhor resultado.

✂️
Seja específico

Um prompt específico gera uma resposta melhor com menos tokens. "Escreva 3 headlines para campanha de lançamento de ração para gatos, tom descontraído" é melhor (e mais eficiente) do que "me ajuda com umas ideias".

🎯
Uma tarefa por conversa

Conversas focadas em uma tarefa específica são mais eficientes do que conversas que misturam vários assuntos. Crie uma nova conversa para cada tarefa diferente: isso também mantém o histórico organizado.

📋
Não cole o que não é necessário

Se você quer análise do parágrafo 3 de um documento, cole só o parágrafo 3: não o documento inteiro. Isso economiza contexto e melhora o foco da resposta.

🔄
Peça refinamento em vez de repetir

Se a resposta ficou perto mas não perfeita, peça refinamento: "Essa headline está boa mas precisa ser mais direta" é mais eficiente do que repetir o prompt inteiro do zero.

💡
Regra prática da LR

Se uma conversa passou de 20 trocas de mensagem ou você colou mais de 3 documentos longos na mesma janela: hora de começar uma nova conversa. Não é perda: é eficiência. Resumo rápido do contexto no início da nova conversa e você retoma em segundos.
